I am going to develop frontend for my College project.

There must be 5 dashboards
	1. Student
	2. Class Advisior
	3. HOD
	4. Warden
	5. Other Dept Incharge.
In each dashboards all requests must be visible..
Only Authenticated users will be allowed to  access their specific routes
I am using google Authentication

1. For student dashboard 
	1.1 Apply for leave
	1.1.2 Fields: Start Date, End Date, Categories: Short, Long, Emergency, Others

	1.2 Apply for Bonafide
	1.2.1 Categories: Internship, General, Educational Loan, Scholarship Bonafide
	1.2.2 For Internship Bonafide - Specify Start Date and End Date

	1.3 Apply for Outpass
	1.3.1 Fields: Out time, Outdate, Contact Number, Parent Contact, Purpose (mandatory)
	1.3.2 Fields: Hostel ID, Floor ID, Room Number, In time,  In date (optional) (mandatory for hostelers)

	1.4 Apply for OD
	1.4.1 Fields:  From date, To date, Purpose, Proof file (pdf) (mandatory)
	1.4.2 Fields: Start time, End time, Place (optional)
	
	
	1.5 Complaint raise
	1.5.1 Fields: File (to upload) (optional), Text (mandatory)

	1.6 Students must be able to track all requests raised and their status

2. For class Advisor dashboard
	2.1 For Leave, Bonafide, Outpass, OD
	2.1.1 For each request - Only 2 options Approved or Rejected

	2.2 Advisor must be able to track all requests they acted on or pending

3. For HOD dashboard
	3.1 For Bonafide, Outpass, OD
	3.1.1 For each request - Only 2 options Approved or Rejected

	3.2 HOD must be able to track all requests they acted on or pending

4. For Warden
	3.1 For Outpass
	4.1.1 For each request - Only 2 options Approved or Rejected
	
	4.2 Warden must be able to track all requests they acted on or pending

5. For other dept heads like (water, electricity) 
	5.1 Only for Complaints
	5.1.1 For each request - Open, In_Progress, Resolved

	5.2 Dept Incharge must be able to track all requests they acted on or pending
